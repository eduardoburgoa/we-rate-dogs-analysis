Wrangle Report¶

A) Introduction¶

I have a analysed a WeRateDogs dataset combining different sources of information: a csv, twitter API and a dog breed predictions table.

The data to analyze is from to tweets like this one: https://twitter.com/dog_rates/status/887473957103951883

In this report I am an going to describe the steps followed in the data Wrangling process.

B) Gathering Data¶

There were 3 different sources:

  1. A csv provided in the project to download.
  2. An URL to obtain a .tsv file with the breed predictions.
  3. Twitter API to obtain missing information.

B.1) Gathering .csv file¶

This was the easiest step. I have downloaded the file. I have uloaded to the same folder as my Jupyter notebook. Finally I have loaded it in a dataframe using pandas read_csv method.

B.2) Gathering .tsv file in a remote URL¶

I used requests.get(url) method to get the content. Then I have openend a local file using mode='wb' and I saved the response.content on this .tsv local file. Finally I have loaded it in a dataframe using pandas read_csv(filename, sep='\t').

B.3) Gathering Twitter API info¶

I have created a twitter developer account following project instructions howto. I have obtained all the keys needed (consumer_key, consumer_secret, access_token, access_secret). I have set up the tweepy api using wait_on_rate_limit=True and wait_on_rate_limit_notify=True. This is important because I have made a lot of queries surpassing api rate limit. In order to create the tweet_json.txt I have used thweepy api.get_status(status_id, tweet_mode='extended') method and I write it to a file with a line for each JSON reponse using json.dumps() serialization: file.write(json.dumps(tweet_status._json))and file.write("\n")

C) Asessing Data¶

I have found the following issues:

Quality Issues¶

Completeness¶

  • Twitter table has 2356 rows and only 2297 expanded url values
  • Twitter table has 2356 rows and image predictions table has 2075 rows.

Validity¶

  • There are some tweets that are replies.
  • There are some tweets that are retwetts.
  • Image predictions table has only 1532 p1 dog predictions.
  • retweeted_status_id has wrong data type. It is float data type and should be integer.
  • retweeted_status_user_id has wrong data type. It is float data type and should be integer.
  • timestamphas wrong datatype. It is a string and should be datetime

Acuracy¶

  • There are duplicated Names as 'None', 'a', 'the' and other lowercase words which not seem valid names for a Dog
  • Image predictions table p2 and p3 predictions have a probability lower of 0.5.
  • Image predictions table has only 973 p1 predictions with a probability greater than 0.5.

Consistency¶

  • Dog stage columns doggo,floofer, pupper, puppo contain 'None' values instead of Nan
  • There are values of rating_numerator below 10 and therefore there are not consistent with We rate dogs rating system.
  • There are values of rating_denominator different from 10 and therefore there are not consistent with We rate dogs rating system.

Tidiness Isssues¶

  • There are three different tables and we only have two types of observational units.
  • Dog stage as is one variable should be only one column. Now is distributed is in several columns doggo, floofer, pupper and puppo. This column should be an enumerate.
